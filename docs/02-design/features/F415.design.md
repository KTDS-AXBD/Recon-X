# F415 Design — LPON bundled R2 fix

## 1. 변경 파일 목록

| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| `services/svc-skill/src/ai-ready/evaluator.ts` | 수정 | loadSpecContent에 r2Key 파라미터 추가 |
| `services/svc-skill/src/ai-ready/evaluator.test.ts` | 수정 | r2Key 파라미터 테스트 추가 |
| `services/svc-skill/src/routes/ai-ready.ts` | 수정 | D1 r2_key 조회 + loadSpecContent 전달, error label 수정 |
| `scripts/upload-bundled-r2.ts` | 수정 | hardcoded cwd 경로 → 자동 감지 |

## 2. evaluator.ts 변경 설계

### Before (현재)
```ts
export async function loadSpecContent(
  env: Env,
  skillId: string,
  _organizationId: string,
): Promise<{ specContent: SpecContent; skillName: string } | null> {
  const r2Key = `skill-packages/${skillId}.skill.json`;
  const obj = await env.R2_SKILL_PACKAGES.get(r2Key);
```

### After (변경 후)
```ts
export async function loadSpecContent(
  env: Env,
  skillId: string,
  _organizationId: string,
  r2Key?: string,           // ← 새 파라미터 (D1에서 전달받음)
): Promise<{ specContent: SpecContent; skillName: string } | null> {
  const key = r2Key ?? `skill-packages/${skillId}.skill.json`;
  const obj = await env.R2_SKILL_PACKAGES.get(key);
  if (!obj) {
    logger.warn("skill package not found in R2", { skillId, r2Key: key });
    return null;
  }
```

- **backward compatible**: 기존 호출부는 파라미터 없이 그대로 동작
- **r2Key 없을 때**: default path `skill-packages/{skillId}.skill.json` (기존 동작 유지)
- **r2Key 있을 때**: D1에 저장된 실제 경로 사용 (bundled skills의 `bundle-` prefix 포함)

## 3. routes/ai-ready.ts 변경 설계

### handleAiReadyEvaluateSingle — skill row 조회 변경

```ts
// Before
const skillRow = await env.DB_SKILL.prepare(
  "SELECT skill_id FROM skills WHERE skill_id = ? AND organization_id = ?",
).bind(skillId, organizationId).first<{ skill_id: string }>();

// After
const skillRow = await env.DB_SKILL.prepare(
  "SELECT skill_id, r2_key FROM skills WHERE skill_id = ? AND organization_id = ?",
).bind(skillId, organizationId).first<{ skill_id: string; r2_key: string }>();
```

### loadSpecContent 호출 변경

```ts
// Before
const loaded = await loadSpecContent(env, skillId, organizationId);
if (!loaded) {
  return notFound("spec-container", skillId);  // ← 잘못된 label

// After
const loaded = await loadSpecContent(env, skillId, organizationId, skillRow.r2_key);
if (!loaded) {
  return notFound("skill-r2", skillId);  // ← 정확한 label
```

## 5. 테스트 계약

### evaluator.test.ts — r2Key 파라미터 테스트

1. `loadSpecContent(env, skillId, org, "skill-packages/bundle-xxx.skill.json")` — bundled 경로로 정상 로드
2. `loadSpecContent(env, skillId, org, undefined)` — 기존 default 경로 fallback 정상 동작
3. R2에 파일 없을 때 null 반환 (기존 동작 유지)

## 6. R2 r2_key 구조 정리

| 상태 | D1 r2_key 형식 | 실제 R2 파일 존재 여부 |
|------|---------------|----------------------|
| draft/published (superseded 전) | `skill-packages/{uuid}.skill.json` | ✅ 존재 |
| bundled | `skill-packages/bundle-{uuid}.skill.json` | ❌ 미존재 (TD-55) |
| superseded | `skill-packages/{uuid}.skill.json` | ✅ 존재 |

Fix 후: evaluator가 D1 r2_key를 사용하면, bundled의 경우 `bundle-{uuid}` 경로로 조회.
→ upload-bundled-r2.ts 실행 후 `bundle-{uuid}` 경로 R2 파일 생성 필요.
