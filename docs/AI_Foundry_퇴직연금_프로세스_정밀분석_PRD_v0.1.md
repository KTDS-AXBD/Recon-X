# AI Foundry -- 퇴직연금 도메인 프로세스 정밀 분석 시스템

## Product Requirements Document (PRD)

Version: v0.1\
작성일: 2026-03-03

------------------------------------------------------------------------

# 1. Project Overview

## 1.1 배경

기존 퇴직연금 프로젝트 산출물(요구사항정의서, 프로세스정의서, ERD,
API정의서, 화면설계서 등)은 엑셀, PPT, PDF, DOC 등 다양한 포맷으로
존재한다.

현재 목표는 다음과 같다:

-   퇴직연금 핵심 프로세스의 정밀 스펙화
-   암묵지 형태의 로직을 구조화
-   온톨로지 기반 메타/용어 표준화
-   Human-in-the-loop 기반 검증 체계 구축

------------------------------------------------------------------------

# 2. Problem Statement

현재 문제점:

1.  설계 문서가 포맷별로 분산되어 있음
2.  핵심 프로세스와 메가 프로세스 구분이 명확하지 않음
3.  용어/메타데이터 표준 부재
4.  프로세스-데이터-요구사항 간 정합성 검증 불가
5.  누락/중복/오버스펙 여부 판단 체계 부재

------------------------------------------------------------------------

# 3. Product Goal

파일 업로드 → 자동 분석 → 핵심 프로세스 추출 → 누락/중복/삭제 후보 도출
→ 사용자 검증 → 개선 루프

이를 통해: - 표준화된 핵심 프로세스 정의 확보 - 온톨로지 기반 지식 모델
구축 - AI 기반 프로세스 진단 시스템 완성

------------------------------------------------------------------------

# 4. Scope

## 4.1 In Scope

-   파일 업로드 (xlsx, pptx, pdf, docx)
-   OCR + 구조 분석
-   프로세스/메서드/조건 추출
-   시퀀스/플로우차트 분석
-   요구사항-프로세스 매핑
-   데이터 플로우 매핑
-   누락/중복/오버스펙 분석
-   Human-in-the-loop UI

## 4.2 Out of Scope (1차 릴리즈)

-   완전 자동 리팩토링
-   코드 자동 생성
-   외부 시스템 실시간 연동

------------------------------------------------------------------------

# 5. Functional Requirements

## 5.1 문서 분석 엔진

-   멀티 포맷 파서
-   OCR 처리 (이미지 포함 문서)
-   계층 구조 인식 (대/중/소)
-   메서드 정의 및 호출 조건 추출

## 5.2 프로세스 분석 엔진

-   시퀀스 다이어그램 해석
-   플로우 차트 관계 분석
-   데이터 흐름 추적
-   요구사항 매핑

## 5.3 핵심 프로세스 식별

-   Core Domain 기준 필터링
-   빈도/의존성 기반 중요도 스코어링
-   온톨로지 기반 관계 분석

## 5.4 진단 기능

-   누락 프로세스 탐지
-   중복 로직 탐지
-   오버스펙 탐지
-   정합성 위반 탐지

## 5.5 Human-in-the-loop UI

-   추출 결과 시각화
-   수정/삭제/보완 제안 기능
-   변경 이력 관리
-   재분석 루프 실행

------------------------------------------------------------------------

# 6. Ontology Design

## 6.1 핵심 개념

-   Process
-   SubProcess
-   Method
-   Condition
-   Requirement
-   DataObject
-   Actor
-   Event

## 6.2 관계 정의

-   Process → hasSubProcess
-   Process → consumesData
-   Process → producesData
-   Requirement → satisfiedBy
-   Method → triggeredBy Condition

------------------------------------------------------------------------

# 7. System Architecture

## 7.1 구성요소

1.  Document Ingestion Layer
2.  Parsing & OCR Layer
3.  Semantic Extraction Layer
4.  Ontology Mapping Layer
5.  Process Diagnosis Engine
6.  HITL Validation UI
7.  Feedback Loop Engine

------------------------------------------------------------------------

# 8. User Flow

1.  파일 업로드
2.  자동 분석 실행
3.  핵심 프로세스 시각화
4.  누락/중복/삭제 후보 표시
5.  사용자 검증
6.  온톨로지 업데이트
7.  재분석

------------------------------------------------------------------------

# 9. Technical Stack (제안)

Backend: - Python - FastAPI - NLP (LLM + Embedding) - Graph DB (Neo4j)

Frontend: - React - D3.js (프로세스 시각화)

Infra: - Containerized Deployment - Vector DB - Object Storage

# 11. Success Metrics

-   문서 구조 추출 정확도 ≥ 85%
-   핵심 프로세스 자동 식별 정확도 ≥ 80%
-   사용자 수정률 감소 추이
-   분석 소요시간 50% 단축
