## 2. Schema Design (`packages/types/src/ai-ready.ts`)

### 2.1 Criterion Enum

```typescript
export const AIReadyCriterion = z.enum([
  'source_consistency',       // 1. 소스코드 정합성
  'comment_doc_alignment',    // 2. 주석·문서 일치
  'io_structure',             // 3. 입출력 구조 명확성
  'exception_handling',       // 4. 예외·에러 핸들링
  'srp_reusability',          // 5. 업무루틴 분리·재사용성
  'testability',              // 6. 테스트 가능성 및 단위테스트 적합성
]);
export type AIReadyCriterion = z.infer<typeof AIReadyCriterion>;
```

### 2.2 Score Schema (단일 기준 점수)

```typescript
export const AIReadyScoreSchema = z.object({
  criterion: AIReadyCriterion,
  score: z.number().min(0).max(1),
  rationale: z.string().min(20).max(800),
  passThreshold: z.literal(0.75).default(0.75),
  passed: z.boolean(),  // score >= passThreshold
});
export type AIReadyScore = z.infer<typeof AIReadyScoreSchema>;
```

### 2.3 Evaluation Schema (skill 단위 집계)

```typescript
export const AIReadyEvaluationSchema = z.object({
  skillId: z.string().uuid(),
  skillName: z.string(),
  criteria: z.array(AIReadyScoreSchema).length(6),
  totalScore: z.number().min(0).max(1),       // 6기준 단순 평균
  passCount: z.number().int().min(0).max(6),  // passed=true 개수
  overallPassed: z.boolean(),                  // passCount >= 4
  modelVersion: z.string(),                    // e.g., "claude-haiku-4-5-20251001"
  evaluatedAt: z.string().datetime(),
  costUsd: z.number().nonnegative(),           // 본 skill 평가 비용
});
export type AIReadyEvaluation = z.infer<typeof AIReadyEvaluationSchema>;
```

### 2.4 총점 산식 확정

- **단순 평균**: `totalScore = sum(criteria.score) / 6`
- **통과 판정**: `overallPassed = (passCount >= 4)` — 6기준 중 4기준 이상 통과 시 AI-Ready PASS
- **가중 평균은 Phase 2에서 재결정** (실측 분포 보고 결정, 현 시점 가중치 근거 불충분)

## 3. LLM Prompt Design (`services/svc-skill/src/ai-ready/prompts.ts`)

### 3.1 Interface

```typescript
export interface PromptInput {
  sourceCode: string;          // Java 소스 (rules/runbooks 참조)
  metadata: {
    provenanceYaml: string;    // provenance.yaml
    contracts: string;         // contracts/*.yaml JSON-serialize
    rules: string[];           // rules/*.json 목록
  };
  skillName: string;           // e.g., "lpon-charge"
}

export function buildPrompt(
  criterion: AIReadyCriterion,
  input: PromptInput,
): string;
```

### 3.2 프롬프트 템플릿 (공통 뼈대)

```
당신은 Java 엔터프라이즈 코드의 AI-Ready 품질을 평가하는 시니어 엔지니어입니다.
아래 skill의 {criterion_korean} 기준 점수를 0~1 사이 숫자로 평가하고,
근거를 소스 line 인용과 함께 150~250단어로 작성하세요.

## 평가 기준: {criterion_korean}
{criterion_definition}

## 평가 가이드
{criterion_rubric}   <- 0.9+/0.75~0.9/0.5~0.75/<0.5 구간별 명시

## 입력
### 소스 코드
```java
{sourceCode}
```

### 메타데이터
- Provenance: {provenanceYaml}
- Contracts: {contracts}
- Rules: {rules}

## 출력 형식 (JSON만 반환, 다른 텍스트 없이)
{"score": 0.XX, "rationale": "..."}
```

### 3.3 6기준별 Rubric 예시 (criterion 1: 소스코드 정합성)

```
### 평가 기준: 소스코드 정합성
Java 소스가 spec-container 메타데이터(rules/contracts)와 얼마나 일치하는가.

### 평가 가이드
- 0.9+: 모든 rule/contract 필드가 소스에 1:1 구현. 드리프트 없음.
- 0.75~0.9: 1~2개 minor 드리프트 (네이밍/주석 수준). 기능 드리프트 없음.
- 0.5~0.75: 3~5개 드리프트 또는 1개 기능 드리프트 (누락 필드).
- <0.5: 다수 드리프트 또는 구조적 불일치 (메서드 시그니처 vs contract).
```

### 3.4 JSON Mode 강제

- Anthropic API: prefill `{"` + stop sequence `}` 활용
- 파싱 실패 시 프롬프트에 예시 JSON 추가 후 재시도 (최대 2회)

### 4.2 실행 흐름

```typescript
// Pseudo-code
async function main() {
  // 1. 비용 가드 — 사전 체크
  const usage = await fetch(`${LLM_ROUTER_URL}/usage?date=${today()}`);
  if (usage.totalUsd >= 30) {
    await askContinueOrAbort();
    process.exit(1);
  }
  if (usage.totalUsd >= 25) console.warn('⚠️ 비용 $25 초과, 잔여 $5');

  // 2. 샘플링
  const tierA = await loadTierASkills(40);    // lpon-* 7 services 분배
  const random = await loadRandomSkills(40);  // pension + giftvoucher
  const skills = [...tierA, ...random];       // 80 skill

  // 3. 평가 (concurrency=5)
  const evaluations: AIReadyEvaluation[] = [];
  let cumulativeCost = 0;
  for (const batch of chunk(skills, 5)) {
    const results = await Promise.all(
      batch.map((skill) => evaluateSkill(skill, model)),
    );
    evaluations.push(...results);
    cumulativeCost += results.reduce((s, r) => s + r.costUsd, 0);

    // 4. 비용 가드 — 실시간 체크
    if (cumulativeCost + usage.totalUsd >= 30) {
      console.error(`❌ 비용 가드 초과, ${evaluations.length} skill 처리 후 중단`);
      break;
    }
  }

  // 5. 리포트 저장
  await writeFile('reports/ai-ready-poc-{date}.json', JSON.stringify({
    executedAt: new Date().toISOString(),
    modelVersion: model,
    totalSkills: evaluations.length,
    totalCostUsd: cumulativeCost,
    evaluations,
  }, null, 2));
}

async function evaluateSkill(skill: SkillMeta, model: string): Promise<AIReadyEvaluation> {
  const criteria: AIReadyScore[] = [];
  for (const criterion of ALL_CRITERIA) {
    const prompt = buildPrompt(criterion, {
      sourceCode: skill.sourceCode,
      metadata: skill.metadata,
      skillName: skill.name,
    });
    const response = await callLlm({
      tier: model === 'opus' ? 'tier1' : 'tier2',
      messages: [{ role: 'user', content: prompt }],
      callerService: 'ai-ready-poc',
    });
    const parsed = safeParseJson(response.content);
    criteria.push({
      criterion,
      score: parsed.score,
      rationale: parsed.rationale,
      passThreshold: 0.75,
      passed: parsed.score >= 0.75,
    });
  }
  const totalScore = criteria.reduce((s, c) => s + c.score, 0) / 6;
  const passCount = criteria.filter((c) => c.passed).length;
  return {
    skillId: skill.id,
    skillName: skill.name,
    criteria,
    totalScore,
    passCount,
    overallPassed: passCount >= 4,
    modelVersion: model,
    evaluatedAt: new Date().toISOString(),
    costUsd: /* sum from LLM responses */ 0,
  };
}
```

### 4.3 샘플링 전략

- **Tier-A 40건** — lpon-charge/budget/gift/payment/purchase/refund/settlement 7 service 균등 분배 (5~6건씩)

## 5. Cost Guard Design

### 5.1 3단계 가드

| 누적 비용 | 동작 |
|-----------|------|
| $0 ~ $24.99 | 정상 실행 + 호출마다 비용 누적 log |
| $25.00 ~ $29.99 | `console.warn('⚠️ 비용 $25 초과')` + 계속 |
| $30.00 이상 | 즉시 중단 + AskUserQuestion (계속/중단/샘플 축소) |

### 5.2 비용 계산 (Haiku 기준 PoC 예상)

- 1 skill × 6 criteria × ~2000 token input + ~300 token output = ~13,800 token/skill
- Haiku: $0.25/1M input + $1.25/1M output
- 1 skill 비용 ≈ $0.003 + $0.0004 = **$0.0034**
- 80 skill × $0.0034 = **$0.27** (예상)
- Opus 교차 10건: 10 × 6 × (($15/1M × 2000) + ($75/1M × 300)) = **$2.85**
- **총 예상 $3.12** — $30 가드 여유 충분

### 5.3 Opus 교차 검증

- `--opus-cross-check 10` 옵션 — 80건 중 10건 랜덤 추출하여 Opus로도 평가
- 리포트에 Haiku vs Opus score |diff| 평균 기재
- |diff| 평균 > 0.2 시 Phase 2는 Opus 사용 권고

```

## 9. Design Tradeoffs

| 결정 | 대안 | 선택 이유 |
|------|------|-----------|
| 단순 평균 산식 | 가중 평균 | PoC 단계 가중치 근거 불충분, Phase 2 실측 보고 재결정 |
| Haiku 기본 | Opus 기본 | 비용 10배 차이, PoC는 샘플링이라 정확도 신뢰구간 확보가 더 중요 |
| 6 기준 독립 호출 | 1 호출에 6 기준 | JSON 출력 복잡도 증가 + 기준 간 bias, 독립 호출이 품질 안정 |
| concurrency=5 | concurrency=1 / =10 | LLM-Router rate limit 고려 중간값 |
| $30 가드 | $50 / $100 | PRD §5.3 명시된 제약 준수 |
| 10% 수기 검증 | 20% / 전체 | 80건 기준 10%(8건)이 현실 작업량, 더 많으면 PoC 취지 초과 |
