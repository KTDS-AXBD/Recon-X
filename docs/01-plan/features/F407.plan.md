### (a) 재선택 근거

DNS 실상 재조사 + Cloudflare 기능 재확인 결과 (c) 대비 (a) 우위:

| 축 | (a) Zone Move | (c) 신규 zone |
|----|---------------|---------------|
| 도메인 비용 | $0 (기존 minu.best 재사용) | ~$10~30/yr 신규 구입 |
| 브랜딩 | `rx.minu.best` 유지 | 포기 |
| 이해관계자 영향 | **Pages cross-account 지원**으로 www/app 유지 가능 (re-verify만) | 없음 |
| 작업 시간 | NS 전환 + Access 재구성 ~30min | 도메인 구입 + 전파 + 재배포 ~3~4h |
| 리스크 | NS 전환 중 단기 downtime (~수분) | 없음 |

**핵심 근거**: Cloudflare **Pages는 cross-account custom domain 지원**. 즉 zone이 KTDS로 이동해도 개인 계정 minu-web Pages 프로젝트의 `www.minu.best` custom domain은 DNS re-verify만 하면 복구됨. Workers Custom Domain만 same-account 제약이 있어 rx.minu.best가 이걸 사용하게 됨.

```
  rx.minu.best  → CF proxy 104.26.3.214      (KTDS 계정 Pages ai-foundry-web)

DMARC/DKIM: 미설정 (이메일 auth 없음, MX 없음과 정합)
```

### Phase 1 — 사전 준비 (사용자 action, 10~20min)


파일 내용을 전달하면 Master가 완전성 검증 + KTDS 계정 import 준비.

**1-2. 관련 서비스 영향 확인**

| 서비스 | 계정 | 영향 | 대응 |
|--------|------|------|------|
| www.minu.best → minu-web.pages.dev | 개인 | Pages Custom Domain 연결 재검증 필요 | Move 후 Dashboard에서 "Recheck DNS" |
| app.minu.best → minu-web.pages.dev | 개인 | 동일 | 동일 |
| api.minu.best → ? | 개인(추정) | Pages/Worker 연결 재검증 | Move 후 re-verify. Worker라면 cross-account 제약 발동 가능 → 별도 조치 |

**1-3. Registrar 확인**: minu.best를 구입한 registrar(Gabia, Namecheap, Cloudflare Registrar 등). NS pair가 이관 후 변경되면 registrar에서 NS 수정 필요.

**1-4. Downtime 창 합의**: NS 전환 중 수 분~수십 분 DNS 일시 불안정 구간. 영향 서비스: www/app/api.minu.best 사용자 접근 일시 실패. 야간(23:00~07:00 KST) 권장.

### Phase 6 — Zero Trust Access Application 재구성 (사용자 action, 15min)

세션 236 구성 재활용 + 드리프트 복구:

```
Dashboard (ktds.axbd@gmail.com) > Zero Trust > Access > Applications
  기존 2건 (Decode-X Public / Protected) 상태 확인
    → Bypass 정책 유지 확인 (경로: /welcome, /assets/*, /favicon.ico, /_routes.json)
    → Allow 정책 유지 확인 (@kt.com Google IdP)
  Application Order: Public 상위, Protected 하위 (Bypass가 먼저 매칭되어야 함)

  드리프트 있으면 재생성:
  6-1. Decode-X Public (Bypass)
       - Type: Self-hosted
       - Application Domain: rx.minu.best
       - Path matching: /welcome, /assets/*, /favicon.ico, /_routes.json
       - Policy: Bypass (Everyone)
  6-2. Decode-X Protected (Allow)
       - Type: Self-hosted
       - Application Domain: rx.minu.best
       - Path: / (전체)
       - Policy: Allow @kt.com via Google IdP
```

### Phase 7 — 검증 (Master 실측, 5min)

**7-1. DNS**
```bash
dig NS minu.best                 # 신규 KTDS NS pair
dig CNAME rx.minu.best            # app-web.ktds-axbd.workers.dev
dig CNAME www.minu.best           # minu-web.pages.dev
dig A api.minu.best               # CF proxy IP
```

**7-2. HTTP 스모크 13건**
```bash
for p in / /welcome /executive /assets /favicon.ico /_routes.json \
         /cdn-cgi/access/authorized /cdn-cgi/access/callback \
         /cdn-cgi/access/login /cdn-cgi/access/logout; do
  CODE=$(curl -s -o /dev/null -w "%{http_code}" "https://rx.minu.best$p")
  echo "$p → $CODE"
done
```

**기대**:
- `/` → 302 (IdP redirect)
- `/welcome` → 200 (Bypass)
- `/executive` → 302
- `/assets/*` → 200
- `/favicon.ico` → 200
- `/_routes.json` → 200
- `/cdn-cgi/access/*` → 302 또는 200 (404 해소가 핵심)

**7-3. 시크릿 창 login flow**
- `https://rx.minu.best/welcome` 접근 → Google 로그인 버튼 클릭 → Google 인증 → `/` 진입 완료

**7-4. 부수 서비스 확인**
- `https://www.minu.best/` → HTTP 200 Next.js 정상
- `https://app.minu.best/` → HTTP 200 동일
