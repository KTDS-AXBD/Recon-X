## 3. 결정 매트릭스 — DO vs KV

| 기준 | Durable Objects | KV (선택) |
|------|----------------|-----------|
| 일관성 | 강한 일관성 | Eventually consistent |
| 비용 | 요청당 $0.4/백만 + 월 $5 | 읽기 $0.5/백만, 쓰기 $5/백만 |
| 복잡도 | DO 클래스 + wrangler binding | 단순 KV put/get |
| HITL 적합성 | 과다 — 동시성 불필요 | 충분 — one-shot resume |
| TTL 자동 만료 | 수동 alarm 필요 | expirationTtl 네이티브 |

**결정: KV** — HITL은 one-shot(한 번 resume 후 세션 폐기)이라 eventual consistency 허용. DO 비용/복잡도 대비 효용 낮음.

## 4. HITL Response Schema

```typescript
// POST /agent/resume request (기존 AgentResumeRequestSchema)
{
  resumeToken: string;       // KV lookup key
  decision: string;          // "approve" | "reject" | "modify"
  data?: Record<string,unknown>; // 수정 파라미터 (optional)
}

// POST /agent/resume response (신규 — nextStep 추가)
{
  status: "resumed";
  resumeToken: string;
  runId: string;
  nextStep: {
    type: "tool_call" | "complete";
    toolName?: string;        // type="tool_call" 시 필수
    args?: Record<string,unknown>;
    summary: string;
  };
}
```

## 5. resumeToken 라이프사이클

```
handleAgentRun()
  → (TOOL_CALL_END 이후 HITL 필요 시)
  → crypto.randomUUID() → resumeToken
  → KV.put("session:{token}", JSON, { expirationTtl: 86400 })
  → SSE emit HITL_REQUEST event { resumeToken }
  → (스트림 종료, RUN_FINISHED 생략)

handleAgentResume(resumeToken)
  → KV.get("session:{token}")
  → null → 404 NOT_FOUND (만료 또는 무효)
  → found → parse AgentSessionState
  → KV.delete("session:{token}") // 1회성
  → nextStep 계산 (decision 기반)
  → HTTP 200 { status, resumeToken, runId, nextStep }
```

## 6. KV 네임스페이스 준비 (배포 전 수동 수행)

```bash
# 개발/기본
wrangler kv:namespace create AGENT_SESSIONS
# → id: <생성된 ID> → wrangler.toml [[kv_namespaces]] id 에 기입

# 프로덕션
wrangler kv:namespace create AGENT_SESSIONS --env production

# 스테이징
wrangler kv:namespace create AGENT_SESSIONS --env staging
```

> **Sprint 252 scope**: wrangler.toml에 binding 선언 포함. 실 namespace ID는 배포 시 `wrangler kv:namespace create` 후 교체 필요. CI는 KV optional(`env.AGENT_SESSIONS?`) 처리로 placeholder ID 없이도 통과.
