# F403 — Phase 9 신규 라우트 E2E 커버리지 보강 설계

## 1. 목표

6개 신규 라우트(F378~F387)의 E2E 커버리지 갭 종결.
Sprint 241에서 구현 완료된 4개 spec 파일의 설계 정합성 확인.

## 2. Spec 파일 설계

### (a) executive-evidence.spec.ts — F378

**대상**: `/executive/evidence` 허브 페이지

| 테스트 | Locator 전략 | 검증 |
|--------|-------------|------|
| heading 렌더 | `getByRole("heading", { name: /근거 자료/ })` | `toBeVisible()` |
| 탭 URL param | `getByRole("tab", { name: /조직 종합 Spec/ })` | `toHaveURL(/tab=org-spec/)` |

**설계 근거**: CF Access 환경에서 `VITE_DEMO_MODE=1` + `?demo=1` bypass 활용 (playwright.config.ts 기본 설정).

### (b) engineer-workbench.spec.ts — F379/F380

**대상**: `/engineer/workbench` SplitView + `/:id` detail 라우트

| 테스트 | Locator 전략 | 검증 |
|--------|-------------|------|
| workbench 렌더 | `getByRole("heading", { name: "Engineer Workbench" })` + `getByPlaceholder("Skill ID 입력...")` | `toBeVisible()` |
| detail URL param | `goto("/engineer/workbench/test-skill-id")` → input 확인 | `toHaveValue("test-skill-id")` |

**설계 근거**: `/:id` param을 URL에서 읽어 input 기본값 설정 — Playwright URL 직접 접근으로 SPA 라우터 파라미터 주입 검증.

### (c) admin.spec.ts 확장 — F382/F387

**대상**: `/admin` 대시보드 + AuditLog 탭 전환

| 테스트 | Locator 전략 | 검증 |
|--------|-------------|------|
| admin 헤딩 렌더 | `getByRole("heading", { name: /Admin 대시보드/ })` | `toBeVisible()` |
| AuditLog 탭 전환 | `getByRole("tablist").getByRole("tab", { name: /감사 로그/ })` | `waitForLoadState("networkidle")` + tabpanel active |

**설계 근거**: Radix UI `Select.Item value=""` 금지 (AuditLog.tsx ALL_ROLES sentinel="all"). tabpanel active 상태로 탭 전환 완료 검증.

### (d) guest-mode.spec.ts — F384

**대상**: `/?demo=guest` read-only 세션 + `/upload` GuestBlockedView

| 테스트 | Locator 전략 | 검증 |
|--------|-------------|------|
| GuestBlockedView | `goto("/?demo=guest")` → `goto("/upload")` → `main` 스코프 | `getByText(/데모 모드에서 사용할 수 없는 기능/)` |
| Demo Mode 배지 | `goto("/?demo=guest")` | `getByText("Demo Mode")` |

**설계 근거**:
- `test.use({ storageState: { cookies: [], origins: [] } })` — engineer storageState 격리
- `main` 스코프 locator — Sidebar와 strict mode 충돌 방지

## 3. CF Access 인증 패턴

모든 spec은 `auth.setup.ts`의 storageState를 사용 (playwright.config.ts `projects[].use.storageState`).
guest-mode만 `test.use({ storageState: { cookies: [], origins: [] } })`로 격리.
