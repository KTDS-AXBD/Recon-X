
Workers ESBuild bundle에서 `import.meta.url` undefined → `createRequire(undefined)` throw → cold-start fail.

## 2. 근본 원인 분석

### 2.1 ESM 파일 (`web-tree-sitter.js`) 문제

```js
var ENVIRONMENT_IS_NODE = typeof process == "object" && process.versions?.node && ...;
if (ENVIRONMENT_IS_NODE) {
    const { createRequire } = await import("module");
    var require = createRequire(import.meta.url);  // ← import.meta.url = undefined in bundle
}
```

CF Workers + `nodejs_compat`: `process.versions.node` = polyfilled → `ENVIRONMENT_IS_NODE = true` → `import.meta.url` 사용 → bundle에서 undefined → crash.

### 2.2 CJS 파일 (`web-tree-sitter.cjs`) 문제 (2종)

**문제 A — ENVIRONMENT_IS_NODE 오감지**:
```js
var ENVIRONMENT_IS_NODE = ... && process.versions?.node ...;
// nodejs_compat이 process.versions.node를 설정 → true
// 그 후 __dirname 사용 → CF Workers에서 __dirname 미정의 → ReferenceError
```

**문제 B — ENVIRONMENT_IS_WORKER 오작동**:
```js
var ENVIRONMENT_IS_WORKER = typeof WorkerGlobalScope != "undefined";
// CF Workers에서 WorkerGlobalScope 존재 → true
// _scriptName = self.location.href → self.location = undefined → TypeError
```

**문제 C — 런타임 WASM 컴파일 금지**:
```js
WebAssembly.instantiate(rawBytes, imports)  // CF Workers: runtime compilation 금지
// CompileError: Wasm code generation disallowed by embedder
```

## 4. 확정 우회 패턴 (Sprint 257 적용 사양)

### 4.1 CJS 파일 patch 2종 (`patch-package`)

```diff
# web-tree-sitter+0.26.8.patch

# Patch 1: ENVIRONMENT_IS_NODE — CF Workers has process.versions.node (polyfill) but no __dirname
- var ENVIRONMENT_IS_NODE = typeof process == "object" && process.versions?.node && process.type != "renderer";
+ var ENVIRONMENT_IS_NODE = typeof process == "object" && process.versions?.node && process.type != "renderer" && typeof __dirname !== "undefined";

# Patch 2: ENVIRONMENT_IS_WORKER — CF Workers self.location is undefined
-   _scriptName = self.location.href;
+   _scriptName = (typeof self !== "undefined" && self.location) ? self.location.href : void 0;
```

### 4.2 wrangler.toml 추가 항목

```toml
[alias]
"web-tree-sitter" = "./path/to/node_modules/web-tree-sitter/web-tree-sitter.cjs"

[[rules]]
type = "CompiledWasm"
globs = ["**/*.wasm"]
fallthrough = true  # avoid DEFAULT rule collision warning
```

### 4.3 WASM 로딩 코드 패턴

```ts
// wrangler [[rules]] type="CompiledWasm" converts .wasm to WebAssembly.Module at build time
// @ts-expect-error — no TS type for CompiledWasm import
import runtimeWasm from "../wasm/web-tree-sitter.wasm";

await Parser.init({
  instantiateWasm(imports: WebAssembly.Imports, receive: (inst: WebAssembly.Instance, mod: WebAssembly.Module) => void) {
    // WebAssembly.instantiate(Module, imports) = instantiation only (no compilation = CF Workers safe)
    WebAssembly.instantiate(runtimeWasm as WebAssembly.Module, imports)
      .then((instance: WebAssembly.Instance) => receive(instance, runtimeWasm as WebAssembly.Module));
  },
} as unknown as EmscriptenModule);
```

## 7. 신규 교훈

1. **emscripten 자동 생성 코드의 CF Workers 환경 감지 오작동**: `nodejs_compat`이 `process.versions.node`를 polyfill하면 `ENVIRONMENT_IS_NODE=true` 오감지 → `__dirname` 사용 → crash. 동시에 `WorkerGlobalScope` 존재로 `ENVIRONMENT_IS_WORKER=true` → `self.location.href` crash.

2. **CF Workers WASM 제약**: 런타임 `WebAssembly.instantiate(bytes)` 금지. 빌드타임 `[[rules]] type="CompiledWasm"` + 런타임 `WebAssembly.instantiate(Module)` 패턴이 필수.

3. **instantiateWasm hook**: emscripten `Module["instantiateWasm"]`은 CF Workers WASM 로딩 우회의 표준 진입점. `receive(instance, module)` 순서 주의.

4. **[alias] wrangler.toml**: ESM/CJS entry 선택을 bundler에서 강제 — emscripten 패키지 Workers 호환성 우회의 표준 패턴으로 정립.
