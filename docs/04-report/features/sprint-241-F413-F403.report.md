## 2. F413 — Skill Packaging Lifecycle 표준화 (TD-53 해소)

### 2.1 문제 요약

`skills.status` enum이 schema(0001_init)에 3종(`draft/published/archived`)만 정의되고 CHECK 제약 없어, 코드에서 3종 추가(`bundled/reviewed/superseded`) drift 발생. `routes/ai-ready.ts:214` SQL filter `status='published'`가 production LPON published 0건과 미스매치 → HTTP 400 차단.

### 2.2 구현 요약

| 파일 | 변경 내용 |
|------|---------|
| `infra/migrations/db-skill/0013_skills_status_check.sql` | 6-enum CHECK 제약 + org/status 복합 인덱스 (CREATE→INSERT→DROP→RENAME 패턴) |
| `services/svc-skill/src/routes/ai-ready.ts:214` | `status='published'` → `status IN ('bundled','reviewed')` |
| `services/svc-skill/src/routes/skills.ts` | `SKILL_STATUSES` 상수 + `UpdateStatusSchema`/`BulkPublishSchema` enum 6개 |
| `services/svc-skill/src/routes/skills.test.ts` | `bundled`/`reviewed`/`superseded` 신규 status 허용 + bulk 테스트 |
