## 신규 발견 (교훈)

1. **`spec-content-adapter.ts:source.excerpt` 해석 오류**: 이 필드는 "원본 발췌"이지만 exception column으로 매핑됨. 문서 의도와 코드 동작 불일치가 0% PASS의 구조적 원인.
2. **단일 시나리오 템플릿 한계**: 기존 template이 policy당 1 scenario 생성 + 추상 표현 → testability rubric 미충족. 3개 concrete scenarios가 필요한 이유.
3. **r2KeyOverride 패턴**: D1 migration 없이 스크립트에서 augmented path를 오버라이드할 수 있어 PoC에 이상적. 전체 시스템 변경 없이 실험 가능.
