# F357 AgentResume 실구현 Design

## 1. AgentSessionState — KV 저장 스키마

```typescript
interface AgentSessionState {
  runId: string;
  organizationId: string;
  task: string;
  nextToolCall: { toolName: string; args: Record<string, unknown> } | null;
  hitlComponentType: "PolicyApprovalCard" | "EntityConfirmation" | "ParameterInput";
  hitlProps: Record<string, unknown>;
  createdAt: number; // Date.now()
}
```

**KV key**: `session:{resumeToken}` (UUID v4)
**TTL**: `expirationTtl: 86_400` (24시간)
**KV binding name**: `AGENT_SESSIONS`

## 2. 시퀀스 다이어그램 — HITL Round-trip

```
FX Orchestrator       svc-mcp-server          KV (AGENT_SESSIONS)
      │                     │                        │
      │ POST /agent/run      │                        │
      │────────────────────>│                        │
      │                     │ generateToken()        │
      │                     │ ─────────────────────> KV.put("session:{token}", state, ttl=86400)
      │<─SSE RUN_STARTED────│                        │
      │<─SSE TOOL_CALL_START│                        │
      │<─SSE TOOL_CALL_END  │                        │
      │<─SSE HITL_REQUEST   │ { resumeToken }        │
      │                     │                        │
      │  [사용자 판단]       │                        │
      │                     │                        │
      │ POST /agent/resume  │                        │
      │  { resumeToken,     │                        │
      │    decision }       │                        │
      │────────────────────>│                        │
      │                     │ KV.get("session:{token}")
      │                     │ ─────────────────────> │
      │                     │ <──── state ───────── │
      │                     │ KV.delete("session:{token}")
      │                     │ ─────────────────────> │
      │<────────────────────│ 200 { status:"resumed",│
      │  nextStep           │   runId, nextStep }    │
```

## 3. KV 네이밍 규약

| 항목 | 값 |
|------|-----|
| key prefix | `session:` |
| key format | `session:{UUIDv4}` |
| value | JSON string (AgentSessionState) |
| TTL | 86,400초 (24h) |
| 1회성 | resume 성공 시 즉시 delete |

## 6. agent.ts 변경 — handleAgentRun

`handleAgentRun(request, env)` 파라미터에 `env: Env` 추가.

HITL 흐름 (SSE 이벤트 순서):
1. `RUN_STARTED`
2. `TOOL_CALL_START` (query-policies)
3. `TOOL_CALL_END` (policies: 6건)
4. `STATE_SYNC` (차트 위젯)
5. `CUSTOM/HITL_REQUEST` (PolicyApprovalCard, `resumeToken`) ← **신규**

> `RUN_FINISHED`는 resume 후 nextStep에서 표현하므로 run 단계에서 생략.

HITL_REQUEST 이벤트 직전:
```typescript
const resumeToken = crypto.randomUUID();
const state: AgentSessionState = { runId, organizationId, task, ... };
if (env.AGENT_SESSIONS) {
  await env.AGENT_SESSIONS.put(`session:${resumeToken}`, JSON.stringify(state), {
    expirationTtl: KV_SESSION_TTL,
  });
}
```

## 7. agent.ts 변경 — handleAgentResume (stub 교체)

```
입력: { resumeToken, decision, data? }

1. env.AGENT_SESSIONS 없음 → ok({ nextStep: { type:"complete", summary:"degraded" } })
2. KV.get("session:{resumeToken}") → null → notFound("session")
3. KV.delete("session:{resumeToken}")
4. nextStep 결정:
   - decision === "approve" && state.nextToolCall → { type:"tool_call", toolName, args, summary }
   - otherwise → { type:"complete", summary:"결정 처리됨 — 워크플로우 종료" }
5. ok({ status:"resumed", resumeToken, runId, nextStep })
```

## 9. 테스트 매트릭스 (agent.test.ts)

| 테스트 | 검증 내용 | Expected |
|--------|----------|---------|
| resume — 유효 토큰 approve | KV에 state 있음 + decision=approve | 200, nextStep.type=tool_call |
| resume — 유효 토큰 reject | KV에 state 있음 + decision=reject | 200, nextStep.type=complete |
| resume — 토큰 없음 | KV.get() → null | 404 |
| resume — KV 미바인딩 | env.AGENT_SESSIONS = undefined | 200, nextStep.type=complete (degraded) |
| resume — 잘못된 body | resumeToken 누락 | 400 |
