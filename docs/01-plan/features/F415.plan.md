# F415 Plan — LPON bundled R2 fix

## 1. 문제 정의

- **증상**: `POST /skills/{id}/ai-ready/evaluate` → HTTP 404 "spec-container not found"
- **대상**: LPON 35개 bundled skills (D1 `status='bundled'`)
- **확인된 사실**: superseded 859건 R2 존재, bundled 35건 R2 NOT_FOUND

## 2. Root Cause 분석

### 원인 A (Primary): evaluator R2 경로 불일치

| 위치 | 경로 |
|------|------|
| `rebundle-orchestrator.ts:145` (저장) | `skill-packages/bundle-{skillId}.skill.json` |
| `evaluator.ts:63` (읽기) | `skill-packages/{skillId}.skill.json` |

`bundle-` prefix 불일치 → evaluator가 항상 NOT_FOUND 반환.

### 원인 B (Secondary): 일부 bundled skills R2 실제 미존재 가능성

rebundle Worker 실행 시 `env.R2_SKILL_PACKAGES.put()` 호출은 코드에 있으나,
프로덕션 실행 시점 오류로 일부가 미업로드된 것으로 추정 (CLAUDE.md 알려진 issue).

## 3. Fix 전략

### Fix 1: evaluator.ts — D1 r2_key 사용 (코드 수정)

`handleAiReadyEvaluateSingle`에서 이미 D1 skill row를 조회하므로,
`r2_key` 컬럼을 함께 읽어 `loadSpecContent`에 전달.

```
routes/ai-ready.ts:
  - SELECT skill_id, r2_key FROM skills
  - loadSpecContent(env, skillId, orgId, r2KeyFromD1)

evaluator.ts:
  - loadSpecContent(env, skillId, orgId, r2Key?: string)
  - r2Key가 있으면 사용, 없으면 default 경로 fallback
```

### Fix 2: scripts/upload-bundled-r2.ts — 경로 수정 (스크립트 수정)

현재 hardcoded cwd path: `/home/sinclair/work/axbd/res-ai-foundry/services/svc-skill`
→ worktree 기준 상대경로로 변경.

### Fix 3: routes/ai-ready.ts:144 — error label 보정

`notFound("spec-container", skillId)` → `notFound("skill-r2", skillId)`
(실제 오류 위치를 정확히 반영)
